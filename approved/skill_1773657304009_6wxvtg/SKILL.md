# PDF查看与处理 Skill

## 用途
当需要读取和分析PDF文件（如简历、文档、报告等）时，使用本Skill指导操作流程。

## 核心经验

### 1. 首选方案：使用 `pdf` 工具
```
pdf:
  pdf: "文件路径"
  prompt: "分析要求"
```

**成功关键：**
- 模型选择很重要！不同模型的PDF处理能力不同
- **推荐模型**：`glm-5` 对PDF解析成功率较高
- 如果当前模型失败，可以尝试切换模型后重试

### 2. 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `engine_overloaded` | 模型引擎过载 | 等待几秒后重试，或切换模型 |
| `Local media path is not under an allowed directory` | 路径权限限制 | 将文件复制到工作区 `clawd/` 目录下 |
| 文件名乱码/无法识别 | 中文编码问题 | 重命名为英文文件名（如 `resume.pdf`） |
| 工具返回空结果 | 模型无法解析 | 尝试用 `read` 工具读取原始内容，或用其他方案 |

### 3. 标准操作流程

#### Step 1: 准备文件
```powershell
# 复制到工作区（如果原路径不允许）
copy "原路径\文件.pdf" "C:\Users\{用户名}\clawd\temp\"

# 重命名为英文（避免编码问题）
rename "中文文件名.pdf" "resume.pdf"
```

#### Step 2: 尝试PDF工具
```
pdf:
  pdf: "C:\Users\{用户名}\clawd\temp\resume.pdf"
  prompt: "详细分析内容，包括：..."
```

#### Step 3: 如果失败，尝试替代方案

**方案A：使用Python提取文本**
```python
import PyPDF2
with open('file.pdf', 'rb') as f:
    reader = PyPDF2.PdfReader(f)
    for page in reader.pages:
        print(page.extract_text())
```

**方案B：使用 `read` 工具（仅限文本PDF）**
```
read:
  path: "file.pdf"
```

**方案C：浏览器打开+截图分析**
```
browser:
  action: open
  url: "file:///path/to/file.pdf"
```
⚠️ 注意：浏览器无法直接打开本地文件，需要上传到云端

### 4. 最佳实践

1. **先尝试，遇阻再查Skill**
   - 不要一上来就查Skill，先直接尝试 `pdf` 工具
   - 遇到具体错误再回查本Skill的解决方案

2. **模型选择**
   - `glm-5`：PDF解析能力强，推荐优先尝试
   - `kimi-k2.5`：通用能力强，但PDF处理可能不稳定
   - 如果当前模型失败，考虑切换模型

3. **文件预处理**
   - 始终将文件放在允许的路径（工作区内）
   - 中文文件名建议重命名为英文
   - 大文件（>10MB）可能需要更长时间

4. **提示词优化**
   - 明确要求结构化输出（分点、表格等）
   - 指定需要提取的关键字段
   - 要求"完整详细，不要遗漏"

### 5. 简历分析专用模板

```
请详细分析这份简历的内容，包括：
1. 个人信息（姓名、年龄、联系方式等）
2. 教育背景（学历、专业、院校）
3. 工作经历（详细列出每段工作的时间、公司、职位、职责）
4. 技能特长（软件、工具、专业能力）
5. 项目经验（主要项目及成果）
6. 其他亮点或问题

请提供完整详细的内容摘要，不要遗漏重要信息。
```

### 6. 错误处理原则

根据 `execution-standards` 的**遇阻停顿原则**：
1. 遇到错误先**暂停**
2. **查阅本Skill**找解决方案
3. **自我尝试**所有可行方法
4. 仍无法解决再**向用户汇报**
5. 提供**2-3个替代方案**
6. 明确给出**推荐方案**
